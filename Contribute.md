# Contributing to URL Shortener

Thank you for considering contributing to this project! Contributions are welcome and appreciated. By contributing, you help make this URL Shortener more robust and feature-rich.

## How to Contribute

1. **Fork the repository**

   Fork the [URL Shortener repository](https://github.com/goelraghav002/url_shortner) by clicking the "Fork" button in the top-right corner of the page.

2. **Clone your forked repository**

   After forking, clone the repository to your local machine.

   ```bash
   git clone https://github.com/<your-username>/url_shortner.git
   ```

3. **Create a new branch**

   Create a new branch for your feature, bug fix, or improvement. Choose a clear and descriptive branch name.

   ```bash
   git checkout -b feature/add-new-feature
   ```

4. **Make your changes**

   Implement your changes in the codebase. Be sure to:

   - Follow the project's coding standards.
   - Ensure that your code is well-documented and easy to understand.
   - Write unit tests where appropriate.

5. **Test your changes**

   Run the tests locally to ensure everything works as expected before submitting your pull request.

   ```bash
   # example command to run tests
   npm test
   ```

6. **Commit your changes**

   Commit your changes with a descriptive commit message.

   ```bash
   git add .
   git commit -m "Add new feature for shortening URLs"
   ```

7. **Push your changes**

   Push your changes to your forked repository.

   ```bash
   git push origin feature/add-new-feature
   ```

8. **Open a pull request**

   Go to your forked repository on GitHub and open a pull request (PR) to the `main` branch of the original repository. Include a clear description of what your PR does and any relevant information about the changes.

9. **Wait for review**

   A project maintainer will review your pull request. You may be asked to make further changes, so be responsive and ready to update your PR.

## Code Style and Guidelines

- Use [ESLint](https://eslint.org/) for linting JavaScript code.
- Follow [JavaScript Standard Style](https://standardjs.com/) (or the style used in this project).
- Write clear and concise comments in your code.
- Keep functions small and focused on a single task.

## Issues and Discussions

If you have found a bug or have an idea for an enhancement, please open an issue on GitHub. Make sure to check if the issue already exists before creating a new one.

- **Bug reports**: Include a detailed description of the problem and steps to reproduce it.
- **Feature requests**: Describe the feature you'd like to add, and explain how it will benefit the project.

## Getting Help

If you need help or have any questions, feel free to reach out by opening a new discussion in the GitHub repository.
